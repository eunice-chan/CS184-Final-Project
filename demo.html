Instructions

  • Reset view: resets view position to the default position.
  • Reset model: resets model position to the default position.
  • Pose model: randomly translates bones to randomly pose model.
  • Toggle line: show/hide the line betweenn the target and end point/effector. Doesn't affect functionality. Helpful for visualizing the distance between the two.
  • Target - follow mouse: sets the XY position of the target to the mouse position.
  • Target - to endpoint: sets the position of the target to the endpoint.
  • Target - near endpoint: sets the position of the target near the endpoint.
  • Target - X/Y/Z position: manually set the XYZ position of the target.
  • Forward Kinematics / Pose - Joint # - Move: set the XYZ position of the joint.
  • Forward Kinematics / Pose - Joint # - Rotate: set the XYZ rotation of the joint.
  • Inverse Kinematics - Update Speed: sets alpha value used to linearly interpolates between calculated parameters and current parameters.
  • Inverse Kinematics - Repeat: run the method's function every frame.
  • Inverse Kinematics - One Step: run the method's function once.
  • Inverse Kinematics - Method: choose the method to use to calculate IK.
  • Inverse Kinematics - Joint # - Move: enable/disable XYZ position of the joint as a parameter for the IK method.
  • Inverse Kinematics - Joint # - Rotate: enable/disable XYZ rotation of the joint as a parameter for the IK method.
  • Inverse Kinematics - Levenberg-Marquardt: aka DLS (damped least squares). Modify parameters related to this method.
    • Max Iter: maximum number of times to fail to find a better set of parameter values before deciding to stick with the current set of parameter values.
    • Lambda: the initial value of lambda.
    • Increment: factor by which to increase lambda.
    • Decrement: factor by which to decrease lambda.
  • Inverse Kinematics - Sequential Monte Carlo: aka particle filter. Modify parameters related to this method.
    • # of Particles: number of samples/particles to take at a time.
    • Distribution: factor of how much the particle is mutated.

Notes

  • Some parameter combinations are difficult for the implemented LM algorithm so it may look like it is frozen. Sometimes it takes a while to find better parameter values, so it acts like it's laggy, and sometimes it just doesn't move, ever. In that case, just move the target to the endpoint and try to guide it to a different position, or just reposition it directly with forward kinematics.
  • With LM, sometimes, the end point/effector doesn't reach the target. This may be because it can't reach from the position it got itself to--that is, it had fallen into a local minimum and can't get out. This algorithm works best when the initial guess is close to the global minimum, or if there is only one minimum. Like other iterative algorithms, LM can get stuck in local minima.
